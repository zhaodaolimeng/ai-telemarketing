# 会话状态发现

## 预期会话状态（假设）

基于催收业务先验知识，我们预期可能存在以下状态，但最终以数据发现为准：

### 1. 开场阶段
| 状态 | 说明 |
|------|------|
| `opening` | 问候、自我介绍 |
| `identity_confirm` | 确认对方身份 |
| `purpose_explain` | 说明来电目的 |

### 2. 信息核实阶段
| 状态 | 说明 |
|------|------|
| `debt_statement` | 说明欠款情况 |
| `debt_acknowledge` | 客户承认/否认 |
| `excuse_listen` | 倾听客户借口 |
| `excuse_response` | 回应客户借口 |

### 3. 施压/协商阶段
| 状态 | 说明 |
|------|------|
| `consequence_remind` | 提醒后果 |
| `empathy` | 共情安抚 |
| `negotiate` | 讨论还款方案 |
| `pressure` | 施加压力 |

### 4. 收尾阶段
| 状态 | 说明 |
|------|------|
| `commitment_extract` | 引导承诺 |
| `commitment_confirm` | 确认承诺 |
| `closing` | 礼貌结束 |

### 5. 特殊状态
| 状态 | 说明 |
|------|------|
| `hostile` | 客户敌对/愤怒 |
| `silent` | 客户沉默 |
| `transfer_request` | 要求转人工 |

## 状态发现步骤

### Step 1: 数据准备
```python
# 提取每个utterance的特征
features = []
for utterance in all_utterances:
    feat = {
        'text_embedding': sentence_bert(utterance.text),
        'speaker': utterance.speaker,
        'position': utterance.turn_idx / total_turns,
        'prev_state': ...,
        'next_state': ...,
        'dialogue_id': utterance.dialogue_id
    }
    features.append(feat)
```

### Step 2: 可视化探索
- 绘制 embedding 的 2D/3D 可视化
- 颜色标注：成功/失败、说话人、位置
- 看是否有明显的簇结构

### Step 3: 聚类
```python
# 使用 HDBSCAN（自动确定簇数）
import hdbscan

clusterer = hdbscan.HDBSCAN(
    min_cluster_size=50,
    min_samples=10,
    prediction_data=True
)
labels = clusterer.fit_predict(embeddings)
```

### Step 4: 人工解读与命名
对每个簇：
1. 随机抽样50-100个样本
2. 阅读文本，找出共同点
3. 尝试命名
4. 讨论并合并相似的簇

输出: `results/states/state_definitions.md`
