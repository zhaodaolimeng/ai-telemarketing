# 黄金数据集标注指南

## 概述
本文档描述如何对印尼语催收对话进行标注，生成黄金测试数据集。该数据集将用于：
1. 对话回放测试，确保机器人优化后不出现功能退化
2. 评估指标校准，确保离线评估结果与真实业务效果一致
3. 模型训练数据，用于提升机器人的应对能力

## 标注流程

### 1. 环境准备
- 确保已经安装Python 3.8+
- 确保代码仓库已经克隆到本地
- 确保`data/processed/transcripts/`目录下有所有对话文件
- 确保`data/gold_dataset_annotation_list.json`文件已经生成
- 所有样本已经经过自动标注工具预标注，标注员只需要复核和修正

### 2. 启动标注工具
```bash
python scripts/annotation_tool.py
```

### 3. 标注步骤
1. 工具启动后会显示当前标注进度，优先展示需要复核的自动标注样本
2. 选择要标注的样本编号（优先选择高优先级的稀有场景样本）
3. 工具会显示完整的对话内容和自动标注结果
4. 复核自动标注的各个字段，修正错误的标注：
   - 重点检查意图标注是否正确
   - 检查对话阶段划分是否合理
   - 检查通话结果判定是否准确
   - 检查合规检查是否正确
5. 确认所有字段无误后保存标注结果

### 4. 批量自动标注
对于新的未标注样本，可以先运行自动标注工具生成预标注结果，再人工复核：
```bash
python scripts/batch_annotate.py
```

## 字段说明

### 基本信息 (basic_info)
| 字段 | 类型 | 说明 | 可选值 |
|------|------|------|--------|
| collection_stage | enum | 催收阶段 | H2: 宽限期前2天（温和引导）<br>H1: 宽限期前1天（引导+暗示后果）<br>S0: 实质性逾期（高压催收） |
| call_duration | number | 通话总时长，单位秒 | 自动填充，无需修改 |
| call_result | enum | 通话结果 | success: 成功拿到还款承诺<br>failure: 催收失败<br>extension: 用户同意展期<br>partial_payment: 用户同意部分还款 |

### 用户画像 (user_profile)
| 字段 | 类型 | 说明 | 可选值 |
|------|------|------|--------|
| persona | enum | 用户画像类型 | cooperative: 合作型，愿意配合还款<br>busy: 忙碌型，现在没空接电话<br>negotiating: 协商型，想要商量还款方案<br>resistant: 抗拒型，不愿意还款<br>silent: 沉默型，很少说话<br>forgetful: 健忘型，不记得有借款<br>excuse_master: 借口型，有各种理由不还款 |
| resistance_level | enum | 用户抗拒程度 | very_low: 非常配合<br>low: 比较配合<br>medium: 中等抗拒<br>high: 比较抗拒<br>very_high: 非常抗拒<br>* 系统会提供建议值，可以根据实际情况调整 |

### 对话轮次 (dialogue)
每个对话轮次需要标注以下字段：
| 字段 | 类型 | 适用角色 | 说明 |
|------|------|----------|------|
| turn_number | number | 所有 | 对话轮次，从1开始，自动填充 |
| speaker | enum | 所有 | 说话人：agent(坐席)/customer(用户)，自动填充 |
| text | string | 所有 | 说话内容，自动填充 |
| stage | enum | 所有 | 对话阶段：<br>greeting: 问候阶段<br>identity: 身份确认阶段<br>purpose: 说明来意阶段<br>ask_time: 询问还款时间阶段<br>push: 施压催促阶段<br>negotiate: 协商还款阶段<br>commit: 确认承诺阶段<br>close: 结束通话阶段 |
| is_correct | boolean | 坐席 | 坐席回复是否正确合理：<br>true: 回复正确，符合催收规范<br>false: 回复不正确，违反规范或有更好的表达方式 |
| standard_response | string | 坐席 | 该场景下的标准回复，当is_correct为false时必须填写 |
| user_intent | string | 用户 | 用户说话的核心意图，简明扼要描述用户想表达的意思 |

### 合规检查 (compliance)
| 字段 | 类型 | 说明 |
|------|------|------|
| has_violation | boolean | 对话中是否有违规内容：<br>true: 有违规<br>false: 无违规 |
| violation_details | string | 违规详情，当has_violation为true时必须填写，描述具体的违规内容 |

### 标注信息 (annotation_info)
| 字段 | 类型 | 说明 |
|------|------|------|
| labeler | string | 标注人姓名 |
| label_time | string | 标注时间，自动填充 |
| notes | string | 其他备注信息，可选 |

## 标注规范

### 1. 通话结果判定标准
- **success**: 用户明确承诺在某个具体时间还款（包括承诺还款金额、部分还款时间）
- **extension**: 用户明确要求延期还款，或者坐席同意用户延期/展期还款
- **partial_payment**: 用户明确同意先还部分欠款，并且给出了部分还款时间
- **failure**: 其他情况，包括用户拒绝还款、用户说没钱、用户挂电话、用户说会考虑但没给具体时间等

### 2. 用户画像判定标准
| 用户类型 | 判定标准 |
|----------|----------|
| cooperative | 用户态度友好，愿意配合，没有表现出抗拒，很快承诺还款 |
| busy | 用户明确说现在很忙、在开会、在开车，让稍后再打过来 |
| negotiating | 用户主动提出想要延期、分期、减免利息，或者跟坐席商量还款方案 |
| resistant | 用户态度恶劣，明确说不还钱，让不要再来电话，或者威胁要投诉 |
| silent | 用户大部分时间不说话，只有简单的嗯啊回应，或者全程沉默 |
| forgetful | 用户说不记得有这笔借款，需要确认，或者问什么时候借的、借了多少 |
| excuse_master | 用户找各种理由不还款：没钱、家人生病、生意失败、工资没发等 |
| scam_suspect | 用户怀疑是诈骗，要求验证身份、要证据、质疑坐席合法性 |
| app_uninstalled | 用户说已经卸载了APP、找不到APP、忘了APP密码，无法通过APP还款 |
| knowledge_seeker | 用户询问很多细节问题：费用构成、逾期后果、还款方式、结清证明等 |
| short_extension_seeker | 用户只要求短期宽限几天（1-7天），不需要正式展期 |

### 3. 对话阶段判定标准
| 阶段 | 判定标准 |
|------|----------|
| greeting | 通话开始时的问候语，比如"你好"、"早上好"、"请问是XXX先生吗" |
| identity | 坐席自我介绍，说明来自哪个公司，确认用户身份的阶段 |
| purpose | 坐席说明来电目的，提醒用户有账单逾期的阶段 |
| ask_time | 坐席询问用户什么时候能够还款的阶段 |
| push | 坐席向用户施压，说明逾期后果、催促用户尽快还款的阶段 |
| negotiate | 双方协商还款方案的阶段，包括讨论延期、分期、减免等 |
| handle_objection | 坐席处理用户异议的阶段，包括回复用户疑问、安抚用户情绪、处理投诉威胁等 |
| commit | 用户做出还款承诺，或者坐席确认用户还款时间的阶段 |
| close | 通话结束前的道别语，比如"谢谢配合"、"再见"等 |

### 4. 坐席回复正确性判定标准
坐席回复不正确的情况包括但不限于：
- 违反合规要求：辱骂用户、威胁用户、泄露用户隐私、虚假承诺等
- 话术不规范：没有使用礼貌用语、没有称呼用户、表达不清晰等
- 逻辑错误：回答用户问题错误、给出错误的信息、前后矛盾等
- 应对不当：面对用户的抗拒或疑问没有采取合适的应对策略

当回复不正确时，需要提供标准的正确回复作为参考。

### 5. 用户意图标注规范
- **必须使用标准意图枚举值**（见下方列表），不要自由描述
- 选择最符合用户核心意图的枚举值
- 参考意图说明匹配场景选择
- 如果没有匹配的意图，选择`unknown`
- 例如：
  - 用户说"我现在很忙，你晚点再打过来" → `busy_later`
  - 用户说"我这个月没钱，能不能下个月再还" → `ask_extension`
  - 用户说"你是谁" → `question_identity`

#### 标准用户意图枚举列表
| 意图枚举值 | 说明 | 触发场景 |
|-----------|------|---------|
| `deny_identity` | 用户否认身份 | 说打错了、不是本人、你找错人了 |
| `busy_later` | 用户现在忙 | 说现在不方便、稍后再说、在开会/开车 |
| `threaten` | 用户威胁投诉/报警 | 说要告到OJK、报警、投诉监管机构 |
| `ask_extension` | 用户申请展期/延期 | 要求延长还款时间、宽限几天、延期还款 |
| `ask_amount` | 用户询问欠款金额 | 问总欠款多少、总费用多少、金额多少 |
| `question_identity` | 用户质疑坐席身份 | 问你是谁、从哪里来、要证据、怀疑是诈骗 |
| `request_identity_verification` | 用户要求验证坐席身份 | 要求提供工作证明、授权书、验证身份合法性 |
| `request_interest_reduction` | 用户要求减免利息/罚息 | 要求减少利息、免除罚息、减免费用 |
| `request_short_extension` | 用户要求短期延期 | 要求宽限几天（3-7天）、下周再还、等几天 |
| `complain_high_interest` | 用户抱怨利息太高 | 说利息太高、费用不合理、比约定的高 |
| `app_uninstalled` | 用户说已经卸载APP | 说APP删了、没装、找不到APP、忘了密码 |
| `request_payment_reminder` | 用户要求发送还款提醒 | 要求发WhatsApp/SMS提醒、发账单详情 |
| `request_settlement_proof` | 用户要求结清证明 | 要还款凭证、结清证明、还款收据 |
| `inquire_consequences` | 用户询问逾期后果 | 问不还会怎么样、有什么影响、逾期后果 |
| `borrowing_money` | 用户说正在筹钱 | 正在借钱、找朋友周转、等工资到账 |
| `transfer_in_process` | 用户说正在转账 | 已经在转了、正在处理、马上转 |
| `no_money` | 用户说没钱还 | 经济困难、工资没发、没钱、还不起 |
| `confirm_time` | 用户给出明确还款时间 | 说具体还款时间、日期、时间点 |
| `agree_to_pay` | 用户同意还款 | 说会还、要处理、同意还款但没给时间 |
| `refuse_to_pay` | 用户明确拒绝还款 | 说不还、不打算还、没钱还 |
| `confirm_identity` | 用户确认身份 | 说是我、对的、应答、确认是本人 |
| `greeting` | 用户问候 | 打招呼、喂、你好、selamat pagi等 |
| `ask_fee` | 用户询问利息/费用 | 问利息多少、罚息多少、手续费多少、为什么这么高 |
| `ask_payment_method` | 用户询问还款方式 | 问怎么还、转账到哪里、要账户信息 |
| `already_paid` | 用户说已经还款 | 说已经还了、刚才转了、已经处理了 |
| `partial_payment` | 用户说要部分还款 | 还一部分、先还一半、分期还、先还点 |
| `third_party` | 第三方接听 | 是家人/同事/朋友、要找的人不在、打错了 |
| `dont_know` | 用户说不知道/听不懂 | 说不知道、不清楚、听不懂、不明白 |
| `unknown` | 无法识别的意图 | ASR识别错误、语义不明确、无法匹配其他意图 |

### 6. 违规判定标准
违规内容包括但不限于：
- 辱骂用户：使用侮辱性、攻击性语言
- 威胁用户：威胁要上门催收、联系家人朋友、爆通讯录、起诉等
- 泄露隐私：在有第三人的场合讨论用户的欠款情况
- 虚假承诺：承诺减免利息、删除不良记录等没有权限的承诺
- 冒充其他机构人员：冒充公检法、银行工作人员等
- 诱导性表述：承诺还款送礼品、返现等
- 不合适时间催收：在晚上9点到早上8点之间催收
- 重复拨打骚扰：短时间内多次拨打用户电话
- 询问无关信息：询问与还款无关的个人信息
- 使用不文明用语：脏话、粗口等

### 7. 语言规范
**所有标注内容必须统一使用英文**，包括但不限于：
- 对话阶段(stage)字段描述
- 用户画像(persona)字段描述
- 用户意图(user_intent)标注
- 违规详情(violation_details)描述
- 备注(notes)字段内容
- 所有标注相关的说明性文字
目的是确保数据集的国际化兼容性，方便后续多语言模型训练和全球团队协作。

## 注意事项
1. 标注时以对话内容为准，不要受系统建议值的影响
2. 对于有歧义的地方，可以在备注中说明
3. 如果发现对话转录错误，可以在备注中说明正确的内容
4. 标注过程中随时可以按Ctrl+C中断，已标注的内容不会自动保存
5. 标注完成后请仔细检查，确认无误后再保存
