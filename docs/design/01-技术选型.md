# 技术选型方案

## 项目定位
- **核心目标**：提供稳定、高效的语音服务（TTS/ASR）
- **语言支持**：印尼文（Bahasa Indonesia）、英文
- **自建优先**：尽量使用开源方案，降低长期成本

---

## 一、TTS（语音合成）方案调研

### 1. VITS / Coqui TTS
- **简介**：基于VITS的高质量语音合成框架，由Coqui开源
- **语言支持**：✅英文 ✅印尼文（可训练）
- **优点**：
  - 音质好，接近真人
  - 多音色支持
  - 支持微调训练
  - 完全开源免费
- **缺点**：
  - 需要GPU加速（推理可CPU但速度慢）
  - 印尼文需要自己训练或找预训练模型
- **推荐指数**：⭐⭐⭐⭐⭐

### 2. ESPnet-TTS
- **简介**：端到端语音处理工具包，包含TTS模块
- **语言支持**：✅英文 ✅印尼文（有预训练）
- **优点**：
  - 学术性强，新算法快
  - 预训练模型丰富
  - 支持多种架构（Transformer、VITS等）
- **缺点**：
  - 配置较为复杂
  - 推理速度一般
- **推荐指数**：⭐⭐⭐⭐

### 3. Edge-TTS
- **简介**：基于Microsoft Edge TTS的非官方API逆向工程
- **语言支持**：✅英文 ✅印尼文（官方支持）
- **优点**：
  - 质量极高，微软品质
  - 完全免费
  - 支持多音色
- **缺点**：
  - 依赖网络请求
  - 可能随时失效（非官方）
- **推荐指数**：⭐⭐⭐（临时方案）

### 4. PaddleSpeech
- **简介**：百度开源的语音工具包
- **语言支持**：✅英文 ✅印尼文（需要训练）
- **优点**：
  - 中文社区支持好
  - 文档齐全
  - 部署相对简单
- **缺点**：
  - 印尼文支持较弱
- **推荐指数**：⭐⭐⭐

### TTS方案建议
**短期方案**：Edge-TTS（快速验证）+ Coqui TTS（自建准备）
**长期方案**：Coqui TTS VITS + 自行训练印尼文模型

---

## 二、ASR（语音识别）方案调研

### 1. OpenAI Whisper
- **简介**：OpenAI开源的通用语音识别模型
- **语言支持**：✅英文 ✅印尼文（直接支持，多语言模型）
- **优点**：
  - 识别准确率非常高
  - 多语言支持极好（99种语言）
  - 推理速度快（优化版）
  - 支持实时流式识别
  - 社区极其活跃
- **缺点**：
  - 模型较大（tiny → large）
  - 大模型需要GPU
- **推荐指数**：⭐⭐⭐⭐⭐（强烈推荐）

### 2. ESPnet-ASR
- **简介**：ESPnet的语音识别模块
- **语言支持**：✅英文 ✅印尼文
- **优点**：
  - 学术领域领先
  - 预训练模型多
  - 支持流式识别
- **缺点**：
  - 配置复杂
  - 性能不如Whisper
- **推荐指数**：⭐⭐⭐

### 3. Wav2Vec 2.0 (Facebook)
- **简介**：Facebook开源的语音预训练模型
- **语言支持**：✅英文 ✅印尼文（可微调）
- **优点**：
  - 预训练效果好
  - 可微调至特定领域
- **缺点**：
  - 不如Whisper易用
  - 需要自己训练下游模型
- **推荐指数**：⭐⭐⭐

### 4. FunASR
- **简介**：阿里开源的语音识别工具包
- **语言支持**：✅英文 ✅印尼文（需训练）
- **优点**：
  - 推理速度快
  - 支持多种部署方式
  - 中文文档好
- **缺点**：
  - 印尼文支持较弱
- **推荐指数**：⭐⭐⭐

### ASR方案建议
**首选方案**：Whisper（tiny/small模型可CPU推理，medium/large需要GPU）
- tiny模型 - 速度快，准确率够用
- small模型 - 平衡速度和准确率
- medium模型 - 较高准确率，推荐印尼文
- large模型 - 最高准确率

## 三、NLU/对话理解方案

### 1. LLaMA / Alpaca 系列
- **简介**：Meta开源的LLM，可微调
- **语言支持**：✅英文 ✅印尼文（可微调）
- **优点**：
  - 理解能力强
  - 可微调
  - 可部署本地
- **缺点**：
  - 需要较大资源
  - 推理速度较慢
- **推荐指数**：⭐⭐⭐⭐

### 2. Mistral / Zephyr
- **简介**：较小但效果好的开源模型
- **语言支持**：✅英文 ✅印尼文（可微调）
- **优点**：
  - 模型小速度快
  - 效果接近大模型
- **缺点**：
  - 需要微调
- **推荐指数**：⭐⭐⭐⭐

### 3. NLU简化方案：意图+实体识别
- **简介**：不使用LLM，使用轻量化方案
- **技术**：
  - FastText / BERT 做分类
  - 规则匹配 + 关键词
- **优点**：
  - 速度极快
  - 资源占用小
  - 可解释性强
- **缺点**：
  - 灵活性较差
- **推荐指数**：⭐⭐⭐⭐⭐（推荐起步方案）

### NLU方案建议
**阶段1**：规则匹配 + 关键词 + 简单分类（快速上线）
**阶段2**：微调小型LLM（Mistral-7B）
**阶段3**：根据效果决定是否需要更大模型

## 四、服务架构方案

### 方案A：Python + FastAPI（推荐）
```
优势：
- AI生态好，TTS/ASR框架都是Python
- 开发速度快
- 社区支持好

技术栈：
- Web框架：FastAPI
- TTS：Coqui TTS
- ASR：Whisper / Faster-Whisper
- NLU：待定
- 消息队列：Redis / RabbitMQ
- 存储：MinIO / OSS（录音）
- 数据库：PostgreSQL
```

### 方案B：Go + Python（混合）
```
优势：
- Go做网关、API层（高并发）
- Python做AI服务（TTS/ASR/NLU）
- GRPC通信

技术栈：
- 网关层：Go + Gin
- AI服务：Python
- 通信：gRPC
- 其他同上
```

### 方案C：全部Python
```
优势：
- 统一语言，维护简单
- 快速迭代
```

### 架构建议
**起步阶段**：方案A（Python + FastAPI，简单）
**中期优化**：可根据需要引入Go做网关

## 五、部署方案

### 本地开发环境
- CPU开发机可以跑 Whisper small + VITS
- 或使用云GPU服务器

### 生产部署
- **方案A**：单台GPU服务器（1-2张GPU卡）
- **方案B**：GPU云服务（AWS/GCP/Azure GPU实例）
- **方案C**：容器化部署 + Kubernetes（大规模）

### 推荐配置
- **入门GPU**：NVIDIA T4 / RTX 3090（1-2张）
- **生产GPU**：NVIDIA A10 / A100（根据并发）

## 六、总体建议方案

### 第一阶段（MVP）
- **TTS**：Edge-TTS（快速）+ Coqui TTS（逐步替换）
- **ASR**：Whisper small/medium（Faster-Whisper优化）
- **NLU**：规则 + 简单分类
- **后端**：FastAPI + Python
- **目标**：验证核心语音能力

### 第二阶段（优化）
- **TTS**：完全迁移到Coqui TTS + 印尼文模型
- **ASR**：优化推理速度
- **NLU**：加入轻量级LLM
- **目标**：提升音质和识别准确率

### 第三阶段（扩展）
- **多语种**：加入更多语言
- **优化**：GPU推理优化，流式处理
- **扩展**：完善NLU能力
