# 红黑对抗训练框架 - Robust Training Framework

## 概述

通过红黑对抗的迭代训练模式，持续提升催收机器人的鲁棒性和核心指标。

---

## 红黑对抗模式设计

### 红方（仿真客户）目标
1. 覆盖已有客户类型（7种）和抗拒程度（5级）
2. 生成更"抗压"的回复
3. 生成更"难以沟通"的场景
4. 探索边界情况（极端抗拒、持续沉默、胡搅蛮缠等）
5. 随机组合多种抗拒策略

### 黑方（催收机器人）目标
1. 在所有场景下保持高成功率
2. 降低最大对话轮数
3. 提升用户体验（流畅度）
4. 识别并应对新的抗拒模式
5. 优化话术，提升说服率

---

## 训练流程

### 第一轮：基准建立
1. 运行当前所有测试场景
2. 记录基准指标（成功率、平均轮数等）
3. 识别薄弱环节

### 第二轮：红方增强
1. 在成功场景基础上，增加1-2级抗拒
2. 加入新的拒绝话术/借口
3. 生成"更难"的测试case
4. 整理新的测试集

### 第三轮：黑方改进
1. 分析失败案例的模式
2. 优化状态机逻辑
3. 补充话术/应对策略
4. 重新测试

### 第四轮：评估
1. 对比基准指标
2. 验证改进效果
3. 记录最佳实践
4. 进入下一轮迭代

## 仿真客户增强策略

### 现有能力（已覆盖）
| 客户类型 | 说明 |
|---------|------|
| cooperative | 合作型 |
| busy | 忙碌型 |
| negotiating | 协商型 |
| silent | 沉默型 |
| forgetful | 健忘型 |
| resistant | 抗拒型 |
| excuse_master | 借口大师 |

### 增强方向
1. **复合类型** - 同时具备多种特征（沉默+抗拒，健忘+协商）
2. **渐进式抗拒** - 对话过程中逐步提升抗拒程度
3. **情绪波动** - 客户态度变化（从合作到抗拒，或反向）
4. **转移话题** - 不直接回答，频繁转移话题
5. **反问攻击** - 反问机器人，质疑机器人
6. **条件承诺** - 提出复杂条件，需要多轮确认
7. **信息缺失** - 忘记/不知道关键信息（身份证、金额等）

## 新增测试场景设计

### 复合类型测试
| 场景 | 描述 |
|------|------|
| 沉默+抗拒 | 先沉默，后拒绝 |
| 健忘+协商 | 健忘客户，但会协商时间 |
| 忙碌+借口 | 忙碌+多个轮换借口 |

### 边界测试
| 场景 | 描述 |
|------|------|
| 极端抗拒型 | 拒绝一切，态度恶劣 |
| 完全沉默型 | 全程不说话 |
| 胡搅蛮缠型 | 逻辑混乱，不断转移 |
| 条件不断变型 | 同意A又变B，不断变更 |

## 核心指标监控

### 成功率指标
- 总体成功率
- 分环节成功率 (H2/H1/S0)
- 分客户类型成功率

### 效率指标
- 平均对话轮数
- 平均对话时长
- 平均TTS调用次数

### 质量指标
- 打断次数（过多打断说明体验差）
- 话术重复率
- 承诺时间落实率

## 训练闭环

```
当前版本
    ↓
全量测试
    ↓
分析失败案例 → 提取模式
    ↓
增强仿真客户 → 生成新case
    ↓
优化催收机器人 → 补充话术/逻辑
    ↓
验证测试 → 对比指标
    ↓
新版本发布
```

---

## 自动化流程

每轮训练输出：
1. 新的测试集（包含新增难case）
2. 失败案例分析报告
3. 优化建议（话术/逻辑）
4. 对比指标报告
