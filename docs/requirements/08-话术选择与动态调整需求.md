# 话术选择与动态调整需求

## 一、话术库设计

### 1.1 话术分类体系

#### 按业务阶段分
| 阶段 | 说明 |
|------|------|
| **开场阶段** | 身份确认、来意说明 |
| **事实陈述阶段** | 欠款金额、逾期时间说明 |
| **施压阶段** | 后果告知、压力施加 |
| **协商阶段** | 还款方案讨论 |
| **承诺确认阶段** | 还款时间金额确认 |
| **结束阶段** | 礼貌收尾、后续提醒 |

#### 按施压强度分
| 强度等级 | 适用场景 | 话术特点 |
|----------|---------|---------|
| **L1 - 轻度** | M0早期、历史好客户 | 友好提醒、强调信用 |
| **L2 - 轻中度** | M0后期、M1早期 | 强调影响、培养意识 |
| **L3 - 中度** | M1中期 | 明确后果、给予压力 |
| **L4 - 中强度** | M1后期、M2早期 | 强调紧迫性、多方施压 |
| **L5 - 强度** | M2+ | 直接沟通后果、时间限制 |

#### 按功能类型分
- **信息传递型**：告知事实、数据
- **询问引导型**：提问获取信息、引导思考
- **共情安抚型**：理解客户情绪、安抚
- **压力施加型**：强调后果、给予压力
- **方案建议型**：提供具体还款建议
- **确认核实型**：确认信息、承诺

### 1.2 话术模板结构
```yaml
话术ID: COLLECTION_001
名称: M0开场_标准
语言: id/en
强度等级: L1
适用场景:
  - 阶段: M0
  - 风险等级: 低
  - 历史标签: 守约
话术内容:
  - 变体1: "Selamat siang, Bapak/Ibu [Nama]. Saya dari [Perusahaan]. Ingin mengingatkan tentang pembayaran pinjaman Bapak/Ibu yang jatuh tempo pada [Tanggal]."
  - 变体2: "Halo Bapak/Ibu [Nama], ini dari [Perusahaan]. Kita ada tagihan yang harus dibayar sebelum [Tanggal] ya."
变量:
  - [Nama]: 客户姓名
  - [Perusahaan]: 公司名称
  - [Tanggal]: 逾期日期/到期日期
合规检查点:
  - 无威胁语言
  - 身份明确
  - 事实准确
```

## 二、话术选择策略

### 2.1 初始话术选择
**选择依据**：
1. **客户画像标签**
   - 风险标签（低/中/高）
   - 意愿标签（强/中/弱）
   - 历史标签（守约/偶尔逾期/习惯性逾期）

2. **业务阶段**
   - M0/M1/M2+
   - 逾期天数
   - 欠款金额

3. **催收历史**
   - 以往有效话术
   - 客户反应模式
   - 承诺兑现情况

**选择算法**：
```
输入：客户标签 + 业务参数 + 历史记录
输出：候选话术列表（按推荐度排序）
规则：
  1. 历史效果好的话术优先
  2. 同标签客户成功率高的话术优先
  3. 近期有效话术优先
  4. 避免使用客户抵触过的话术
```

### 2.2 上下文感知的话术选择
**考虑因素**：
- 对话历史中的客户反应
- 已经说过的内容（避免重复）
- 当前对话目标
- 客户情绪状态

**上下文连贯性要求**：
- 保持话题一致性
- 呼应客户之前的表述
- 避免逻辑跳跃
- 适时回顾之前的共识

## 三、动态调整机制

### 3.1 实时调整触发条件

#### 基于客户态度的调整
| 客户态度变化 | 调整策略 |
|-------------|---------|
| **友好→抵触** | 降低施压强度、增加共情 |
| **抵触→敌对** | 暂停施压、先安抚情绪 |
| **敌对→缓和** | 逐步回到主题、给予台阶 |
| **冷漠→感兴趣** | 把握机会、引导到协商 |

#### 基于客户意图的调整
| 检测到的意图 | 调整策略 |
|-------------|---------|
| **想协商** | 切换到协商话术分支 |
| **想承诺** | 立即引导确认具体时间金额 |
| **找借口** | 针对借口类型选择应对话术 |
| **质疑** | 切换到证据出示/解释模式 |
| **拒绝沟通** | 尝试不同角度、或者礼貌结束 |

#### 基于对话进度的调整
| 对话阶段 | 调整要点 |
|---------|---------|
| **开场不顺** | 调整破冰方式、不要急于进入主题 |
| **关键信息传递失败** | 换一种表达方式、确认是否理解 |
| **协商陷入僵局** | 提出新的方案、或者暂时缓一缓 |
| **承诺模糊** | 引导具体化、不要接受模糊承诺 |

### 3.2 话术变体选择
**同一意思多种表达**：
- 避免重复让客户烦
- 不同表达效果不同
- 记录哪种表达最有效

**变体选择策略**：
1. **随机选择**（初期探索）
2. **基于历史效果**（后期优化）
3. **基于客户特征**（个性化）
4. **避免近期用过的变体**（防厌烦）

## 四、话术组合与流程控制

### 4.1 话术流程模板
```yaml
流程ID: FLOW_M0_STANDARD
名称: M0标准流程
阶段序列:
  1. 开场与身份确认
     - 话术: COLLECTION_001 (M0开场)
     - 检查点: 是否本人
  2. 事实陈述
     - 话术: COLLECTION_010 (欠款说明)
     - 检查点: 客户是否确认
  3. 轻度施压
     - 话术: COLLECTION_020 (信用影响提醒)
     - 检查点: 客户反应
  4. 引导还款
     - 话术: COLLECTION_030 (询问还款时间)
     - 检查点: 是否获得承诺
  5. 确认与结束
     - 话术: COLLECTION_040 (承诺确认)
     - 检查点: 承诺明确
分支规则:
  - 如果客户否认 → 跳转到异议处理分支
  - 如果客户愤怒 → 跳转到安抚分支
  - 如果客户承诺 → 跳转到确认分支
```

### 4.2 话术衔接策略
**自然过渡**：
- 避免生硬切换
- 用过渡句连接
- 呼应客户的话

**过渡句示例**（印尼文）：
- "Seperti yang Bapak/Ibu tahu..."（正如您所知...）
- "Nah, tentang itu..."（关于这个...）
- "Oke, lalu yang penting adalah..."（好的，接下来重要的是...）
- "Kalau begitu, kita..."（那样的话，我们...）

## 五、话术优化与学习

### 5.1 效果追踪
**每个话术记录**：
- 使用次数
- 成功次数（定义：达成当前阶段目标）
- 客户平均反应（正面/中性/负面）
- 打断率
- 最终催回率关联

**效果指标**：
```
话术成功率 = 达成阶段目标次数 / 使用次数
客户接受度 = (正面反应数 + 中性反应数) / 总反应数
最终贡献度 = 使用该话术最终催回的案件数 / 总使用数
```

### 5.2 A/B测试机制
**测试设计**：
- 同一客户群随机分配不同话术
- 对照组使用旧话术
- 实验组使用新话术
- 指标：成功率、客户反应、最终催回率

**自动优胜劣汰**：
- 持续表现差的话术自动下架
- 表现好的话术增加使用频率
- 新话术先小范围测试，再推广

### 5.3 话术生成建议
**基于成功案例**：
- 分析高成功率话术的共同点
- 提取有效句式和表达
- 生成新的候选话术
- 人工审核后上线

## 六、多语言支持

### 6.1 印尼文话术特点
**语气把握**：
- 使用敬语（Bapak/Ibu）
- 语气尊重但专业
- 避免过于直接的表达
- 适当使用当地常用表达

**文化适配**：
- 考虑印尼文化习惯
- 宗教因素的敏感性
- 适当的礼貌用语
- 避免可能冒犯的表达

### 6.2 英文话术特点
**语气把握**：
- 专业、直接但礼貌
- 清晰、简洁
- 避免歧义
- 保持一致的专业形象

### 6.3 话术一致性
**同一语义的多语言版本**：
- 确保意思一致
- 保持相同的施压强度
- 尊重各自语言的表达习惯
- 效果可对比

## 七、话术管理与维护

### 7.1 话术生命周期
```
创作 → 审核 → 测试 → 上线 → 监控 → 优化/下架
```

**各阶段要求**：
- **创作**：基于策略、参考成功案例
- **审核**：合规检查、质量检查
- **测试**：小范围A/B测试
- **上线**：逐步放量
- **监控**：实时效果追踪
- **优化/下架**：根据数据决策

### 7.2 话术版本管理
- 记录每次修改
- 保留历史版本
- 支持版本回滚
- 版本效果对比

## 八、应急与降级话术

### 8.1 异常场景话术
| 场景 | 应对话术原则 |
|------|-------------|
| **客户情绪失控** | 先安抚、不刺激、给予空间 |
| **客户威胁/辱骂** | 保持专业、不争执、适时结束 |
| **客户声称要投诉** | 认真倾听、记录、提供正确渠道 |
| **客户有特殊困难** | 表示理解、引导协商、不轻易承诺 |

### 8.2 系统降级时的话术
**模型理解失败时**：
- "Maaf, saya kurang memahami. Bisa diulangi?"（对不起，我没太理解，可以再说一遍吗？）
- "Mohon jelaskan lebih jelas lagi."（请说得更清楚一点）
- 必要时转人工

**网络/系统问题时**：
- "Maaf, ada gangguan teknis. Kita akan hubungi kembali nanti ya."（对不起，有技术故障，我们稍后再联系）
