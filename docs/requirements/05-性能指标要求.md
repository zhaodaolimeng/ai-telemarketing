# 性能指标要求

## 一、响应时间指标

### 1.1 ASR性能
| 指标 | 目标值 | 说明 |
|------|--------|------|
| 首字延迟 | ≤ 200ms | 从开始说话到第一个字输出 |
| 实时率 | < 0.3x | 处理时间 / 音频时长 |
| 字错误率 | < 10% | 印尼文/英文识别准确率 |
| 延迟模式 | 流式输出 | 不等到整句 |

### 1.2 理解性能
| 指标 | 目标值 | 说明 |
|------|--------|------|
| 理解延迟 | ≤ 100ms | 从收到文本到理解完成 |
| 意图准确率 | ≥ 85% | 意图识别准确率 |
| 实体准确率 | ≥ 90% | 实体提取准确率 |

### 1.3 TTS性能
| 指标 | 目标值 | 说明 |
|------|--------|------|
| 首字延迟 | ≤ 200ms | 从开始生成到第一个字 |
| 生成速度 | ≥ 20字/s | 文字生成速度 |
| 音质 | ≥ 4.0 MOS | 语音质量评分 |
| 流式输出 | 支持 | 边生成边播放 |

### 1.4 端到端延迟
| 指标 | 目标值 | 说明 |
|------|--------|------|
| 用户停→机器人说 | ≤ 300ms | 停止说话到机器人开始回应 |
| 完整理解→开始说 | ≤ 500ms | 完全理解到开始说 |
| 对话回合延迟 | ≤ 1s | 完整一个回合 |

---

## 二、并发性能

### 2.1 并发处理
| 指标 | 目标值 | 说明 |
|------|--------|------|
| 支持并发 | ≥ 100路 | 同时支持的通话路数 |
| 扩缩能力 | 线性扩展 | 增加资源线性提升 |
| 连接建立 | ≤ 1s | 建立通话连接 |

### 2.2 资源占用
| 资源 | 目标值 | 说明 |
|------|--------|------|
| GPU利用率 | 80-90% | 高峰值GPU使用率 |
| GPU显存 | ≤ 80% | 每张卡显存占用 |
| CPU使用率 | ≤ 70% | 平均CPU使用率 |
| 内存占用 | ≤ 80% | 系统内存使用 |

---

## 三、稳定性指标

### 3.1 可用性
- **服务可用性**：≥ 99.5%
- **故障恢复**：≤ 5min
- **热备切换**：无感知

### 3.2 错误处理
- **恢复能力**：自动从错误中恢复
- **降级策略**：故障时的降级方案
- **重试机制**：智能重试策略
- **超时处理**：合理的超时处理

---

## 四、质量指标

### 4.1 语音质量
- **TTS质量**：MOS ≥ 4.0
- **ASR准确率**：≥ 90%
- **无杂音**：无明显噪音和故障

### 4.2 对话质量
- **任务完成率**：≥ 70%
- **转人工率**：≤ 30%
- **打断自然度**：真人评价 ≥ 4.0
- **用户体验**：真人评价 ≥ 4.0

---

## 五、成本指标

### 5.1 资源成本
- **GPU小时成本**：可控
- **每路成本**：可接受范围
- **资源利用率**：优化

### 5.2 优化方向
- **模型量化**：减少资源占用
- **模型蒸馏**：小模型达到大模型效果
- **缓存复用**：减少重复计算
