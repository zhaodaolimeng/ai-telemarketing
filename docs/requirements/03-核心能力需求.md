# 核心能力需求

## 一、超低延迟反馈

### 1.1 延迟要求
- **首包响应时间**：≤ 300ms（从用户停止说话到TTS开始播放）
- **全句延迟**：≤ 500ms（从用户停止说话到完整理解并开始回复）
- **实时体验**：用户感受到接近真实对话的流畅度

### 1.2 优化策略
- **模型分层**：
  - 状态检测用轻量模型
  - 内容理解用中等模型
  - 复杂推理用大模型
- **流式处理**：
  - ASR流式输出，边听边识别
  - 理解流式处理，不等到整句结束
  - TTS流式输出，边生成边播放
- **预测性处理**：
  - 根据对话历史预判用户意图
  - 提前准备候选回复

---

## 二、智能打断能力

### 2.1 打断检测
- **VAD（语音活动检测）**：实时检测用户是否在说话
- **打断意愿识别**：区分用户只是清嗓子 vs 真的想打断
- **紧急程度判断**：用户插话的紧急程度评估

### 2.2 打断策略
- **可打断时机**：
  - 句子之间停顿
  - 语义段落结束
  - 非关键内容时
- **不可打断时机**：
  - 重要信息播报中
  - 确认关键问题时
  - 刚开场自我介绍时
- **打断响应模式**：
  - 立即停止，礼貌接话
  - 缓冲1-2秒，找合适点停
  - 短时重要信息先讲完再停

### 2.3 打断后的处理
- **礼貌接话**："您说"、"请讲"等
- **理解用户输入**：快速理解打断内容
- **上下文衔接**：与上文内容自然衔接
- **继续对话**：处理完打断后回到主流程

---

## 三、实时流式对话

### 3.1 流式ASR
- **实时转录**：用户说话时同步输出文字
- **中间结果**：提供多个候选结果，实时修正
- **延迟优化**：
  - 首字延迟 < 200ms
  - 逐字/逐词输出，不是等到整句
  - 动态修正识别结果

### 3.2 流式理解
- **增量解析**：收到部分内容就开始理解
- **意图预判**：不用等整句，提前判断意图
- **实体提取**：边识别边提取实体信息

### 3.3 流式TTS
- **流式输出**：生成一点播放一点
- **首字延迟**：TTS首字输出 < 200ms
- **自然停顿**：在合适位置停顿，方便用户打断

---

## 四、对话状态管理

### 4.1 状态检测
- **静默检测**：用户是否在说话
- **语气分析**：用户的情绪、态度
- **打断信号**：用户是否想插话
- **注意力检测**：用户是否在认真听

### 4.2 状态模型
- **说话状态**：机器人在说
- **聆听状态**：机器人在听
- **思考状态**：机器人在理解
- **暂停状态**：临时停顿
- **过渡状态**：状态切换中

### 4.3 状态切换
- 流畅切换，无生硬感
- 状态冲突处理
- 异常状态恢复
