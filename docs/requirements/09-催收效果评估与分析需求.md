# 催收效果评估与分析需求

## 一、核心评估指标体系

### 1.1 催回率指标（核心业务指标）
| 指标 | 定义 | 计算方式 | 目标值 |
|------|------|---------|--------|
| **M0催回率** | M0阶段成功催回的比例 | 催回案件数 / M0总案件数 | ≥ 85% |
| **M1催回率** | M1阶段成功催回的比例 | 催回案件数 / M1总案件数 | ≥ 60% |
| **M2+催回率** | M2+阶段成功催回的比例 | 催回案件数 / M2+总案件数 | ≥ 35% |
| **整体催回率** | 全阶段总催回率 | 总催回案件数 / 总案件数 | ≥ 65% |
| **金额催回率** | 催回金额占比 | 催回总金额 / 欠款总金额 | ≥ 70% |

### 1.2 承诺相关指标
| 指标 | 定义 | 目标值 |
|------|------|--------|
| **承诺获取率** | 获得明确承诺的对话占比 | ≥ 75% |
| **承诺兑现率** | 承诺后实际还款的比例 | ≥ 60% |
| **按时兑现率** | 在承诺时间内还款的比例 | ≥ 45% |
| **部分兑现率** | 还了一部分但未还清的比例 | ≤ 20% |

### 1.3 对话效率指标
| 指标 | 定义 | 目标值 |
|------|------|--------|
| **平均通话时长** | 每通电话的平均时长 | 3-5分钟 |
| **接通率** | 接通电话数 / 拨打次数 | ≥ 60% |
| **本人接通率** | 本人接听的比例 | ≥ 50% |
| **平均拨打次数** | 每个案件的平均拨打次数 | ≤ 5次 |
| **转人工率** | 需要转人工的比例 | ≤ 30% |

## 二、对话质量评估

### 2.1 对话完整性评估
**检查点**：
- ✅ 身份是否确认
- ✅ 欠款信息是否准确传达
- ✅ 客户是否理解
- ✅ 是否尝试获取承诺
- ✅ 结束是否礼貌专业

**完整性评分**（0-100分）：
```
完整性 = (完成的检查点数 / 总检查点数) × 100
```

### 2.2 对话流畅度评估
**评估维度**：
- 状态切换自然度（打断、静默处理）
- 上下文连贯性
- 响应及时性
- 话术适配度
- 客户反应自然度

**流畅度评分**（真人评估 1-5分）：
- 5分：非常流畅，像真人对话
- 4分：比较流畅，小瑕疵不影响
- 3分：一般，能完成任务但不够自然
- 2分：较差，明显机械感
- 1分：很差，无法正常对话

### 2.3 合规性评估
**实时检查项**：
- ❌ 暴力、威胁语言
- ❌ 冒充公检法
- ❌ 骚扰无关人员
- ❌ 隐私泄露
- ❌ 非工作时段频繁拨打

**合规评分**：
- 100分：无任何违规
- 0分：严重违规（需立即中断）

**拨打频率分析**：

## 四、客户行为分析

### 4.1 还款行为模式

- 承诺后多久还款
- 周末/工作日还款差异
- 月初/月末还款差异

**还款金额模式**：
- 一次性还清 vs 分期
- 部分还款的金额分布
- 还款金额与承诺金额的对比

### 4.2 对话反应模式
**态度变化追踪**：
- 初始态度 → 最终态度

- 哪些话术会恶化态度

**常见反应模式**：
| 反应模式 | 占比 | 最佳应对策略 |

| 抵触敌对 | 20% | 先安抚、再寻找机会 |
| 直接拒绝/失联 | 10% | 升级处理、多方施压 |

| 标签类型 | 预测准确率 | 说明 |
|---------|-----------|------|
| 风险标签 | ?% | 风险-低/中/高 |
| 意愿标签 | ?% | 意愿-强/中/弱/无 |
| 态度标签 | ?% | 态度-友好/中性/抵触/敌对 |
| 结果标签 | ?% | 承诺/拒绝/协商等 |

## 六、归因分析

### 6.1 成功案例归因
**分析框架**：
```
成功催回 = 哪些因素起了关键作用？
  - 客户本身因素？（有还款能力/意愿）
  - 策略选择正确？（施压强度合适）
  - 话术效果好？（某句话打动了客户）
  - 拨打时机好？（客户心情好/有空）
  - 随机因素？（正好发工资了）
```

**归因方法**：
- 对比相似成功/失败案例
- 分析关键转折点
- 提取成功要素

### 6.2 失败案例归因
**失败原因分类**：
| 失败原因 | 占比 | 可优化空间 |
|---------|------|-----------|
| 客户确实无还款能力 | ?% | 协商方案优化 |
| 客户还款意愿极低 | ?% | 施压策略优化 |
| 策略/话术不当 | ?% | 策略/话术改进 |
| 失联/无法接通 | ?% | 拨打策略/信息更新 |
| 其他 | ?% | - |

**根因分析**：
- 5-Why分析法
- 决策路径回溯
- 关键错误点识别

## 七、A/B测试平台

### 7.1 实验设计
**支持的实验类型**：
- 话术A/B测试
- 策略A/B测试
- 拨打时间A/B测试
- 多变量测试

**实验配置**：
```yaml
实验ID: EXP_001
名称: 话术变体测试
假设: 新话术A比旧话术B成功率高10%
分组:
  - 对照组: 旧话术 (50%)
  - 实验组: 新话术 (50)
分流键: 客户ID（确保同一客户始终在同一组）
样本量: 每组至少1000个案例
显著性水平: 95%
结束条件:
  - 达到样本量
  - 或结果已显著
```

### 7.2 实验分析
**分析指标**：
- 主要指标：催回率
- 次要指标：客户接受度、对话时长、转人工率
- 安全性指标：投诉率、合规问题

**统计检验**：
- 显著性检验（p-value）
- 置信区间
- 效果大小（Effect Size）

## 九、反馈闭环与系统优化

### 9.1 反馈收集
**反馈来源**：
- 实际还款数据（最真实的反馈）
- 人工标注/评分
- 人工坐席的反馈
- 客户投诉/建议
- A/B测试结果

### 9.2 自动优化机制
**优化触发**：
- 某话术持续表现差 → 自动降低权重/下架
- 某策略效果好 → 自动增加使用
- 发现更优拨打时间 → 自动调整

**学习闭环**：
```
数据收集 → 分析 → 发现问题/机会 → 调整策略/话术 → 验证效果 → 标准化 → 继续收集
```

### 9.3 人工审核与干预
**需要人工审核的情况**：
- 自动优化的重大变更
- 新话术上线前
- 异常案例分析
- 策略重大调整

## 十、成本效益分析

### 10.1 成本计算
**成本构成**：
- 计算资源成本（GPU/服务器）
- 人工成本（审核、坐席）
- 通信成本（话费）
- 开发维护成本

**单路成本**：
- 每路通话每分钟成本
- 每个案件平均成本
- 每催回1元的成本

### 10.2 效益计算
**直接效益**：
- 催回的总金额
- 相比纯人工的成本节约
- 相比其他方案的效果提升

**ROI分析**：
```
ROI = (增加的催回金额 - 系统成本) / 系统成本
```
