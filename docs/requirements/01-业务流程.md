# 业务流程

## 总体业务流程

### 1. 任务创建阶段
1. 催收人员/管理员创建外呼任务
2. 配置外呼策略、时间、话术
3. 导入或选择客户名单
4. 设置任务优先级和配额

### 2. 外呼执行阶段
1. 系统根据策略分配机器人
2. 机器人开始外呼客户
3. 根据通话情况进行智能对话
4. 实时记录通话状态和结果

### 3. 对话处理阶段
1. **接听处理** - 客户接听后进行身份确认
2. **意向识别** - 识别客户还款意愿、困难等
3. **协商处理** - 与客户协商还款计划
4. **信息确认** - 确认客户信息、联系方式等
5. **结束通话** - 总结并礼貌结束

### 4. 结果归档阶段
1. 自动记录通话结果和标签
2. 保存完整录音和对话文本
3. 生成通话分析报告
4. 需要人工跟进的转人工处理

## 核心对话流程

### 标准催收对话流程

```
开始
  ↓
[身份确认]
  ├─ 是本人 → 继续
  └─ 非本人/转接 → 结束或请求转接
  ↓
[来意说明]
  - 表明身份
  - 说明来电目的
  ↓
[欠款确认]
  - 确认欠款金额
  - 确认还款日
  ↓
[还款意愿评估]
  ├─ 承诺还款 → [协商还款计划]
  ├─ 困难 → [了解困难]
  └─ 拒绝 → [施压协商]
  ↓
[方案确认]
  - 确认还款时间
  - 确认还款金额
  ↓
[礼貌结束]
  - 感谢
  - 提醒
结束
```

### 特殊场景处理

#### 场景1：客户不承认
- 先出示合同/单据信息
- 提示后果影响
- 仍不承认则礼貌结束，标记后续跟进

#### 场景2：客户情绪激动
- 先安抚情绪
- 表示理解
- 等客户平静后再沟通

#### 场景3：客户有还款意愿但困难
- 了解具体困难
- 协商可行的方案
- 确认方案可行性

#### 场景4：电话不通/无人接听
- 记录拨打状态
- 稍后重拨（按策略）
- 尝试其他联系方式（如有）
