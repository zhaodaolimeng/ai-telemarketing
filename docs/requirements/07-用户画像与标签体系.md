# 用户画像与标签体系需求

## 一、用户画像构建

### 1.1 静态画像（基础信息）
| 维度 | 字段 | 说明 |
|------|------|------|
| **个人信息** | 年龄、性别、职业 | 基本特征 |
| **借款信息** | 借款金额、期数、利率 | 原始合同信息 |
| **还款历史** | 历史还款记录、逾期记录 | 历史行为 |
| **征信情况** | 征信查询记录、他行逾期 | 信用状况 |
| **联系方式** | 主电话、备用电话、地址 | 联系信息 |
| **催收记录** | 历史催收记录、结果 | 催收历史 |

### 1.2 动态画像（实时更新）
| 维度 | 更新频率 | 说明 |
|------|---------|------|
| **当前状态** | 实时 | M0/M1/M2... |
| **欠款情况** | 每日 | 当前欠款、费用、逾期时间 |
| **还款意愿** | 每次通话后 | 根据对话评估 |
| **接听习惯** | 每次通话后 | 什么时候接电话概率高 |
| **常用借口** | 每次通话后 | 客户常说什么理由 |

## 二、标签体系设计

### 2.1 风险标签
#### 还款能力
- `能力-强`：收入高、资产好
- `能力-中`：收入一般、可周转
- `能力-弱`：收入低、明显困难
- `能力-未知`：信息不足

#### 还款意愿
- `意愿-强`：主动还款、历史守约
- `意愿-中`：提醒后会还、偶尔逾期
- `意愿-弱`：习惯性逾期、催了才还
- `意愿-无`：故意拖欠、拒绝沟通

#### 综合风险
- `风险-低`：刚逾期、历史好、能力强
- `风险-中`：逾期一段时间、但还有可能性
- `风险-高`：逾期久、意愿弱、可能坏账

### 2.2 对话标签
#### 态度标签
- `态度-友好`：配合、客气
- `态度-中性`：正常沟通、不冷不热
- `态度-抵触`：不耐烦、想挂断
- `态度-敌对`：愤怒、恶语相向

#### 沟通状态标签
- `状态-在听`：认真倾听
- `状态-分心`：好像在忙别的
- `状态-打断`：频繁打断、急于表达
- `状态-沉默`：不说话、不知道听没听

#### 借口类型标签
- `借口-没钱`：说没钱、穷
- `借口-忘记`：说忘记了、没注意
- `借口-周转`：资金周转困难
- `借口-生病/意外`：个人或家庭出事
- `借口-不承认`：说没借过、不认识
- `借口-其他`：其他理由

### 2.3 催收结果标签
#### 直接结果
- `结果-承诺还款`：明确承诺还款时间金额
- `结果-部分承诺`：承诺还部分，或模糊承诺
- `结果-协商成功`：协商好分期/延期方案
- `结果-拒绝`：明确表示不还/没钱
- `结果-无人接听`：电话没接
- `结果-非本人`：是别人接的
- `结果-挂断`：接通了但被挂断

#### 后续标记
- `标记-需要跟进`：后续还需要再打电话
- `标记-转人工跟进`：复杂情况建议人工
- `标记-承诺兑现提醒`：到时间提醒客户兑现
- `标记-策略调整`：下次换个策略

## 三、标签应用场景

### 3.1 对话策略选择
根据标签决定对话策略：

```
风险高 + 意愿弱 = 高强度施压
风险中 + 意愿中 = 适中施压 + 协商
风险低 + 意愿强 = 提醒为主
态度敌对 = 先安抚 + 控制节奏
态度友好 = 直接沟通 + 要承诺
```

### 3.2 话术选择
- 不同标签的客户用不同话术模板
- 同一意思不同表达，适配客户类型
- 历史有效话术优先使用

### 3.3 拨打策略
- 什么时候打最容易接
- 一天内打几次
- 什么情况放弃
- 什么情况升级处理

### 4.1 内部数据

- 历史还款数据
- 历史催收记录
- 录音和文字记录

### 4.2 实时对话提取
- 实时ASR转文字
- 意图和情感分析
- 关键信息提取
- 对话行为分析

### 4.3 反馈数据
- 后续还款情况
- 人工标注结果
- A/B测试数据
