## 核心评测指标

### 1. 业务效果指标 (Business Metrics)

| 指标名称 | 定义 | 计算方式 | 方向 |
|---------|------|---------|------|
| **任务成功率** (Success Rate) | 成功获取客户明确还款承诺的对话比例 | 成功对话数 / 总对话数 × 100% | ↑ 越高越好 |
| **承诺获取率** (Commitment Rate) | 客户给出具体还款时间的比例 | 获取时间承诺的对话数 / 总对话数 × 100% | ↑ 越高越好 |
| **对话完整度** (Completeness) | 走完完整流程的对话比例 | 到达CLOSE状态的对话数 / 总对话数 × 100% | ↑ 越高越好 |
| **早期还款率** (Early Payment Rate) | S0客户中获取承诺的比例 | S0成功数 / S0总数 × 100% | ↑ 越高越好 |

### 2. 对话质量指标 (Dialogue Quality)

| 指标名称 | 定义 | 评测方式 |
|---------|------|---------|
| **状态转移准确率** | 状态机按预期转移的比例 | 人工标注 / 规则检查 |
| **话术自然度** | 话术是否自然流畅 | 人工评分 (1-5分) |
| **策略正确性** | 话术与环节匹配度 | 人工评分 (1-5分) |
| **槽位填充准确率** | 时间提取的准确率 | (正确提取时间数 / 总提取数) × 100% |

### 3. 系统性能指标 (System Performance)

| 指标名称 | 定义 | 计算方式 | 目标 |
|---------|------|---------|------|
| **端到端时延** (E2E Latency) | 客户说完到系统开始播放的时间 | P50, P90, P99 | ≤ 2s (P90) |
| **ASR准确率** (CER) | 语音识别字符错误率 | 编辑距离 / 总字符数 | ≤ 15% |
| **TTS自然度** | 语音合成质量 | MOS评分 | ≥ 3.5 |
| **系统稳定性** | 无崩溃运行时间 | 在线时长统计 | ≥ 99.5% |
| **并发处理能力** | 同时支持的对话路数 | 压力测试 | ≥ 100路 |

### 4. 成本指标 (Cost Metrics)

| 指标名称 | 定义 | 计算方式 |
|---------|------|---------|
| **每通电话成本** | 单次对话的平均成本 | (基础设施 + API费用) / 通话数 |
| **人力替代率** | 替代人工坐席的比例 | 机器人处理量 / 总处理量 × 100% |
| **ROI** | 投资回报率 | (催回增量 - 系统成本) / 系统成本 |

## 评测方法详解

### 1. 自动化评测 (Auto-Evaluation)

#### 适用指标
- 任务成功率
- 对话完整度
- 槽位填充准确率
- 状态转移准确率
- 时延统计

#### 评测流程

```python
def auto_evaluate(dialogue_log):
    """
    自动化评测函数
    """
    results = {}

    # 1. 检查是否成功
    results['success'] = dialogue_log.success and dialogue_log.commit_time

    # 2. 检查流程完整度
    expected_states = [INIT, GREETING, IDENTIFY, PURPOSE,
                       ASK_TIME, COMMIT_TIME, CONFIRM, CLOSE]
    actual_states = [turn.state for turn in dialogue_log.turns]
    results['completeness'] = calculate_state_coverage(expected_states, actual_states)

    # 3. 验证时间提取准确率
    if dialogue_log.commit_time:
        customer_utt = find_utterance_with_time(dialogue_log.turns)
        extracted_time = dialogue_log.commit_time
        results['slot_accuracy'] = validate_time_extraction(customer_utt, extracted_time)

    return results
```

### 3. 人工评测 (Human Evaluation)

#### 评测人员要求
- 印尼语母语者
- 了解催收业务
- 3-5名评测员

#### 评测维度 (1-5分量表)

| 维度 | 5分 | 3分 | 1分 |
|-----|-----|-----|-----|
| **自然度** | 非常自然，和真人无差别 | 基本自然，偶有不流畅 | 很生硬，机器人感强 |
| **策略适配** | 完美匹配当前场景 | 基本合适，有小问题 | 策略完全不合适 |
| **服务态度** | 礼貌且专业 | 礼貌但生硬 | 态度差，不礼貌 |
| **效率** | 高效获取信息 | 一般，有冗余 | 低效，啰嗦 |

#### 信度计算
- 使用 **Fleiss' Kappa** 计算评测者间一致性
- 目标 Kappa ≥ 0.6 (良好一致性)

## 评测实施流程

### 完整评测周期

```
第1天  准备阶段
  ├─ 部署评测环境
  ├─ 加载Golden Set
  └─ 配置评测工具

第2-3天  自动化测试
  ├─ 运行200+60条用例
  ├─ 收集自动评测数据
  └─ 生成初步报告

第4-5天  人工评测
  ├─ 3-5名评测员打分
  ├─ 计算一致性
  └─ 汇总结果

第6天  分析与报告
  ├─ 多维数据分析
  ├─ 错误归类分析
  └─ 生成评测报告
```

### 持续评测机制

| 评测类型 | 频率 | 内容 |
|---------|------|------|
| **冒烟测试** | 每次提交 | 20条核心用例 |
| **周度评测** | 每周 | 完整200条用例 |
| **月度评测** | 每月 | 200条+人工评测 |
| **季度评测** | 每季 | 完整评测+成本分析 |

## 总结

本评测Benchmark提供了智能催收对话系统的全方位评估框架，包含：

1. **4大维度** - 业务效果、对话质量、系统性能、成本
2. **260条测试用例** - 覆盖18种场景组合+边缘场景
3. **3种评测方法** - 自动、半自动、人工评测
4. **阶段目标** - 短中长期目标设定

通过持续运行本Benchmark，可以系统化地跟踪系统进步，确保最终达到或超越真人坐席的效果。
