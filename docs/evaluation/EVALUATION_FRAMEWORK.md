# 🎯 智能催收机器人评估体系
针对原规则仿真完全无法模拟真实场景的问题，本体系设计了从低成本离线评估到高可信度上线验证的全流程评估方案，覆盖所有业务场景。

---

## ❌ 原有规则仿真的核心问题
1. **真实度极低**：仅能覆盖预设的happy path，完全无法模拟真实用户的多样性、不可预测性和对抗性
2. **场景覆盖不全**：无法覆盖边缘case（全程沉默、恶意辱骂、质疑身份、信号不好等）
3. **结果参考性弱**：评估出来的成功率和线上真实表现相关性低于30%
4. **无法模拟语音链路问题**：纯文本逻辑无法覆盖ASR识别错误、TTS可懂度、打断时机错误等真实通话问题

### 🔹 方案1：真实行为模式驱动的生成式客户模拟器
**推荐指数**：⭐⭐⭐⭐⭐
**落地成本**：中
**真实度**：90%
**适用阶段**：全阶段，替代原规则仿真

#### 核心优势
基于你现有1130条真实对话数据训练的专属用户模拟器，生成的用户行为和真实用户几乎无差异，远优于通用LLM模拟器。

#### 实现逻辑
1. **第一步：真实行为特征挖掘**
   从历史对话数据中提取所有用户特征：
   - 不同逾期阶段用户的常用回复、抗拒点、借口库
   - 印尼本地口语习惯、俚语、各地口音特征
   - 不同类型用户的对话逻辑（沉默用户、易怒用户、借口大师的行为模式）
   - 真实用户常见的转移话题、讨价还价、质疑身份等对抗性行为

2. **第二步：生成专属用户模型**
   基于挖掘到的真实特征，通过Prompt工程让LLM生成的用户完全符合业务特征，而不是凭空生成：
   ```prompt
   你现在是一个[抗拒等级：1-5级]的印尼逾期用户，特征：
   - 逾期阶段：[H2/H1/S0]
   - 逾期天数：[X]天
   - 欠款金额：[X]印尼盾
   - 用户画像：[职业/收入/地域/历史还款记录]
   - 行为模式：[沉默型/易怒型/健忘型/借口大师型/协商型]
   - 常用借口：[没钱/工资没发/在外地/不想还/已经还了]

   回复要求：
   1. 使用印尼口语，符合当地用户的表达习惯
   2. 不会轻易承诺还款，需要催收员多次施压才可能同意
   3. 会主动提出疑问、转移话题、甚至质疑催收员身份
   4. 每次回复控制在1-2句话，符合真实通话的长度
   5. 不要暴露你是AI，要像真实用户一样对话
   ```

3. **第三步：可控参数调节**
   支持10+维度的参数精确控制，可生成任意类型的用户场景：
   - 抗拒等级（1=非常配合，5=完全拒绝）
   - 口音类型（爪哇语/苏门答腊语/巴厘岛口音等）
   - 背景噪音（安静/马路/市场/多人说话）
   - 说话风格（快/慢/清晰/模糊）
   - 常见行为（打断/沉默/转移话题/讨价还价）

#### 评估流程
1. 批量生成不同类型的用户场景（可自定义数量，最多支持数万次）
2. 机器人和模拟器自动完成完整对话流程
3. 自动统计成功率、平均轮数、关键节点应对准确率等指标
4. 自动识别bad case，生成优化建议

### 🔹 方案2：真实对话回放测试
**推荐指数**：⭐⭐⭐⭐⭐
**落地成本**：低
**真实度**：95%
**适用阶段**：版本迭代效果验证

#### 核心优势
完全基于真实历史对话数据，评估结果和线上真实表现相关性高达90%以上，是最具参考性的离线评估方案。

#### 实现逻辑
1. **构建黄金测试集**：
   从历史数据中筛选覆盖所有场景的真实对话（建议≥200条），保留用户的每一句回复作为固定输入：
   - 覆盖所有三个逾期阶段（H2/H1/S0）
   - 覆盖所有用户类型（配合/抗拒/沉默/协商等）
   - 覆盖所有常见对话路径（成功/失败/展期）

2. **回放测试流程**：
   ```
   输入：历史对话中用户的第一句话 → 机器人生成回复
   → 输入历史对话中用户的第二句话 → 机器人生成回复
   → 依次走完整个对话流程
   ```

3. **自动效果评估**：
   对比机器人和人工催收的表现：
   - 最终结果匹配度（是否同样拿到还款承诺）
   - 关键节点应对合理性（面对拒绝/疑问时的回复是否正确）
   - 话术匹配度（是否符合用户阶段和特征）
   - 平均对话轮数对比

### 🔹 方案3：离线合成对照评估
**推荐指数**：⭐⭐⭐⭐
**落地成本**：中
**真实度**：90%
**适用阶段**：上线前效果预测，零业务风险

#### 核心优势
不需要上线、不需要等回款周期，就能准确预测机器人的实际业务效果，和真实AB测试的相关性高达90%以上，完全零风险。

#### 实现逻辑
基于因果推断的「合成对照法」，用历史人工催收数据模拟真实上线效果：
1. **特征匹配**：把机器人测试过的用户，和历史上人工催收过的用户做1:1精准匹配：
   - 匹配维度：逾期阶段、欠款金额、用户画像、历史还款记录、地域、职业等所有特征
   - 匹配算法：倾向性得分匹配（PSM）

2. **效果对比**：对比相同特征的用户，机器人催收和人工催收的最终回款率差异
3. **统计检验**：用显著性检验排除随机因素干扰，得到机器人的真实提升效果

#### 输出结果
- 机器人相对人工的回款率提升/下降比例
- 95%置信区间，判断效果是否显著
- 不同用户群体的效果差异（哪类用户适合机器人，哪类不适合）

### 🔹 方案4：端到端多模态语音仿真评估
**推荐指数**：⭐⭐⭐⭐
**落地成本**：中高
**真实度**：100%
**适用阶段**：上线前全链路测试

#### 核心优势
完全模拟真实电话通话场景，发现纯文本评估完全发现不了的语音全链路问题。

#### 测试范围
不只是做文本层面的对话评估，而是完整模拟整个语音通话链路：
1. **用户端模拟**：用带印尼各地口音、带背景噪音（马路/市场/家里）、不同语速/音量的语音合成模拟用户说话
2. **异常场景模拟**：用户中途打断、说话模糊、信号不好、语速过快/过慢等真实场景
3. **全链路测试**：完整走通「用户语音输入→ASR识别→机器人处理→TTS合成回复→播放」的整个流程

#### 评估维度
| 指标 | 要求 | 说明 |
|------|------|------|
| ASR识别准确率 | ≥90% | 尤其是方言、口音、噪音下的表现 |
| 打断时机准确率 | ≥95% | 会不会抢话、会不会用户还没说完就停止播放 |
| TTS可懂度 | ≥95% | 用户能不能听清、语速合不合适、有没有口音问题 |
| 端到端响应延迟 | ≤300ms | 从用户说完话到机器人开始回复的时间 |
| 整体通话成功率 | ≥90% | 完整完成对话的比例 |

### 🔹 方案5：对抗性鲁棒性评估
**推荐指数**：⭐⭐⭐⭐
**落地成本**：低
**真实度**：85%
**适用阶段**：合规性测试，风险边界验证

#### 核心优势
专门测试催收场景最容易出风险的边界case，避免上线后出现合规投诉。

#### 测试用例分类
自动生成各种极端、恶意、高风险的用户回复，批量测试机器人的应对能力：
1. **恶意对抗类**：「你再打我就报警」「我要投诉你骚扰」「你们是骗子吧」
2. **质疑身份类**：「你是谁？我凭什么相信你？」「你工号多少？让你们领导来跟我说」「把你们公司资质发我看看」
3. **逻辑陷阱类**：「你说我欠了100万，你把凭证发过来我看看」「我上次已经还了，你们怎么还来要？」「你利率超过法定标准了，我要告你们」
4. **极端抗拒类**：全程沉默、只说「没钱」「别打了」「我不还你能怎么样」
5. **信息错误类**：「你打错了，我不是XXX」「我从来没借过你们的钱」

#### 评估输出
- 机器人应对准确率
- 合规风险率（有没有出现违规话术、承诺不符合规定的内容）
- 系统崩溃率（遇到异常输入会不会出错）

### 🔹 方案6：实时效果预测模型
**推荐指数**：⭐⭐⭐
**落地成本**：高
**真实度**：85%
**适用阶段**：线上迭代提速

#### 核心优势
把原来需要等1-2个月的回款评估周期，缩短到当天就能出结果，迭代速度提升几十倍。

#### 实现逻辑
用历史对话数据和对应的最终回款结果，训练一个效果预测模型：
- 输入：机器人的完整对话日志、用户特征、催收阶段、话术类型等
- 输出：这个用户的还款概率（0-100%）
- 准确率：≥85%，和最终实际回款率高度相关

#### 适用场景
- 快速迭代版本：每次优化完机器人，跑几千次测试，当天就能知道效果提升还是下降
- 实时质量监控：上线后每通对话结束就能自动判断质量，不需要等回款

### 🔹 方案7：线上A/B测试
**推荐指数**：⭐⭐⭐⭐⭐
**落地成本**：高
**真实度**：100%
**适用阶段**：最终上线验证

#### 核心优势
最权威的效果验证方式，用真实业务数据验证机器人的实际价值。

#### 实施流程
1. **流量分配**：10%流量走机器人，90%流量走人工坐席，或者不同版本机器人之间分流量对比
2. **核心对比指标**：
   | 指标 | 说明 | 目标值 |
   |------|------|--------|
   | 回款率 | 催收后X天内用户实际还款的比例 | 不低于人工的80% |
   | 投诉率 | 用户投诉比例 | ≤0.1% |
   | 人均效率 | 每天可以处理的通话数量 | 是人工的3-5倍 |
   | 平均通话时长 | 完成一次催收的时间 | ≤90秒 |
3. **显著性检验**：需要足够大的样本量（至少数千通话）确保统计显著性

## 📊 核心评估指标体系
### 业务效果指标
| 指标 | 说明 | 优秀值 |
|------|------|--------|
| 最终承诺还款率 | 用户明确同意还款的对话占比 | ≥40% |
| 展期率 | 用户申请延期还款的占比 | ≤25% |
| 失败率 | 用户明确拒绝/无明确回复的占比 | ≤35% |
| 平均对话轮数 | 完成一次催收需要的交互次数 | ≤6轮 |
| 平均通话时长 | 完成一次催收的通话时长 | ≤90秒 |
| 回款率 | 催收后实际还款的用户比例 | 不低于人工的80% |

### 体验合规指标
| 指标 | 说明 | 要求 |
|------|------|------|
| 打断准确率 | 正确识别用户说话并及时停止播放的比例 | ≥95% |
| 话术匹配准确率 | 话术和用户画像、阶段的匹配程度 | ≥90% |
| 合规率 | 没有出现违规话术（威胁、辱骂、泄露隐私等） | 100% |
| 信息准确率 | 还款金额、日期等关键信息播报正确率 | 100% |
| 投诉率 | 用户投诉比例 | ≤0.1% |

### 技术性能指标
| 指标 | 说明 | 要求 |
|------|------|------|
| 端到端响应延迟 | 从用户说完话到机器人开始回复的时间 | ≤300ms |
| ASR识别准确率 | 语音转文本的正确率 | ≥90% |
| TTS可懂度 | 用户能听清回复的比例 | ≥95% |
| 系统可用性 | 服务正常运行时间比例 | ≥99.9% |
