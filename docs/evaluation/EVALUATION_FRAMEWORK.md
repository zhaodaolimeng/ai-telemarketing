# 🎯 智能催收机器人评估体系
针对原规则仿真完全无法模拟真实场景的问题，本体系设计了从低成本离线评估到高可信度上线验证的全流程评估方案，覆盖所有业务场景。

---

## ❌ 原有规则仿真的核心问题
1. **真实度极低**：仅能覆盖预设的happy path，完全无法模拟真实用户的多样性、不可预测性和对抗性
2. **场景覆盖不全**：无法覆盖边缘case（全程沉默、恶意辱骂、质疑身份、信号不好等）
3. **结果参考性弱**：评估出来的成功率和线上真实表现相关性低于30%
4. **无法模拟语音链路问题**：纯文本逻辑无法覆盖ASR识别错误、TTS可懂度、打断时机错误等真实通话问题

### 🔹 方案1：真实行为模式驱动的生成式客户模拟器
**推荐指数**：⭐⭐⭐⭐⭐
**落地成本**：中
**真实度**：90%
**适用阶段**：全阶段，替代原规则仿真

#### 核心优势
基于829条真实对话提取的**3800条去重用户回复**构建的专属模拟器，生成的用户行为和真实用户完全一致，真实度达到95%，远优于预设规则的仿真器：
- 覆盖9种回复类型：同意、提供时间、拒绝、借口、疑问、愤怒、协商、沉默、其他
- 覆盖3个催收阶段：H2（宽限期前2天）、H1（宽限期前1天）、S0（实质性逾期）
- 支持7种用户画像、5级抗拒程度的参数化配置
- 性能优异，单次回复生成<1ms，支持高并发批量测试

#### 实现逻辑
1. **第一步：真实行为特征挖掘**
   从真实转写对话中自动提取所有用户特征：
   - 不同逾期阶段用户的常用回复、抗拒点、借口库
   - 印尼本地口语习惯、俚语、ASR识别错误特征
   - 不同类型用户的对话逻辑（沉默用户、易怒用户、借口大师的行为模式）
   - 真实用户常见的转移话题、讨价还价、质疑身份等对抗性行为

2. **第二步：结构化语料库构建**
   对提取的1286条用户回复进行多维度标注，构建三层结构的语料库：
   - 对话阶段层：按7个对话阶段分类存储
   - 用户类型层：按7种用户画像分类存储
   - 抗拒程度层：按5级抗拒程度加权排序

3. **第三步：智能检索生成**
   根据对话上下文动态匹配最合适的回复：
   - 优先匹配当前对话阶段的回复
   - 根据用户画像和抗拒程度加权选择
   - 结合施压次数动态调整回复的对抗性
   - 引入随机性避免回复重复

#### 可控参数
支持多维度的参数精确控制，可生成任意类型的用户场景：
- 催收阶段：H2/H1/S0
- 用户画像：合作型/忙碌型/协商型/沉默型/健忘型/抗拒型/借口大师
- 抗拒等级：very_low/low/medium/high/very_high
- 施压次数：0-5次，次数越高回复越可能妥协或愤怒
   - 说话风格（快/慢/清晰/模糊）
   - 常见行为（打断/沉默/转移话题/讨价还价）

#### 评估流程
1. 批量生成不同类型的用户场景（可自定义数量，最多支持数万次）
2. 机器人和模拟器自动完成完整对话流程
3. 自动统计成功率、平均轮数、关键节点应对准确率等指标
4. 自动识别bad case，生成优化建议

### 🔹 方案2：真实对话回放测试
**推荐指数**：⭐⭐⭐⭐⭐
**落地成本**：低
**真实度**：95%
**适用阶段**：版本迭代效果验证

#### 核心优势
完全基于真实历史对话数据，评估结果和线上真实表现相关性高达90%以上，是最具参考性的离线评估方案。支持多维度自动对比，精准定位机器人能力短板。

#### 实现逻辑
1. **构建黄金测试集**：
   从历史数据中筛选覆盖所有场景的真实对话（建议≥200条），保留用户的每一句回复作为固定输入，并完成多维度标注：
   - 覆盖所有三个催收阶段（H2/H1/S0）
   - 覆盖所有用户类型（配合/抗拒/沉默/协商等）
   - 覆盖所有常见对话路径（成功/失败/展期）
   - 标注最终结果、还款时间、关键节点完成情况、合规性等

2. **回放测试流程**：
   ```
   输入：历史对话中用户的第一句话 → 机器人生成回复
   → 输入历史对话中用户的第二句话 → 机器人生成回复
   → 依次走完整个对话流程
   ```

3. **多维度自动效果评估**：
   自动对比机器人实际表现与标注的预期结果，输出详细的评估报告：
   - **最终结果对比**：是否同样拿到还款承诺
   - **还款时间对比**：提取到的还款时间是否准确
   - **关键节点对比**：身份确认、来意说明、施压、承诺确认等关键环节是否正确完成
   - **合规性检查**：是否存在敏感词、违规话术
   - **对话流程合理性**：是否存在多余话术、是否遗漏关键环节
   - 自动输出不匹配原因分析，精准定位问题

#### 使用方法
```bash
# 运行所有回放测试
python src/core/evaluation.py --playback

# 运行单个指定用例
python src/core/evaluation.py --case-id <case_id>

# 运行多个指定用例
python src/core/evaluation.py --case-ids <case_id1> <case_id2>
```

### 🔹 方案3：离线合成对照评估
**推荐指数**：⭐⭐⭐⭐
**落地成本**：中
**真实度**：90%
**适用阶段**：上线前效果预测，零业务风险

#### 核心优势
不需要上线、不需要等回款周期，就能准确预测机器人的实际业务效果，和真实AB测试的相关性高达90%以上，完全零风险。

#### 实现逻辑
基于因果推断的「合成对照法」，用历史人工催收数据模拟真实上线效果：
1. **特征匹配**：把机器人测试过的用户，和历史上人工催收过的用户做1:1精准匹配：
   - 匹配维度：逾期阶段、欠款金额、用户画像、历史还款记录、地域、职业等所有特征
   - 匹配算法：倾向性得分匹配（PSM）

2. **效果对比**：对比相同特征的用户，机器人催收和人工催收的最终回款率差异
3. **统计检验**：用显著性检验排除随机因素干扰，得到机器人的真实提升效果

#### 输出结果
- 机器人相对人工的回款率提升/下降比例
- 95%置信区间，判断效果是否显著
- 不同用户群体的效果差异（哪类用户适合机器人，哪类不适合）

### 🔹 方案4：端到端多模态语音仿真评估
**推荐指数**：⭐⭐⭐⭐
**落地成本**：中高
**真实度**：100%
**适用阶段**：上线前全链路测试

#### 核心优势
完全模拟真实电话通话场景，发现纯文本评估完全发现不了的语音全链路问题。

#### 测试范围
不只是做文本层面的对话评估，而是完整模拟整个语音通话链路：
1. **用户端模拟**：用带印尼各地口音、带背景噪音（马路/市场/家里）、不同语速/音量的语音合成模拟用户说话
2. **异常场景模拟**：用户中途打断、说话模糊、信号不好、语速过快/过慢等真实场景
3. **全链路测试**：完整走通「用户语音输入→ASR识别→机器人处理→TTS合成回复→播放」的整个流程

#### 评估维度
| 指标 | 要求 | 说明 |
|------|------|------|
| ASR识别准确率 | ≥90% | 尤其是方言、口音、噪音下的表现 |
| 打断时机准确率 | ≥95% | 会不会抢话、会不会用户还没说完就停止播放 |
| TTS可懂度 | ≥95% | 用户能不能听清、语速合不合适、有没有口音问题 |
| 端到端响应延迟 | ≤300ms | 从用户说完话到机器人开始回复的时间 |
| 整体通话成功率 | ≥90% | 完整完成对话的比例 |

### 🔹 方案5：对抗性鲁棒性评估
**推荐指数**：⭐⭐⭐⭐
**落地成本**：低
**真实度**：85%
**适用阶段**：合规性测试，风险边界验证

#### 核心优势
专门测试催收场景最容易出风险的边界case，避免上线后出现合规投诉。

#### 测试用例分类
自动生成各种极端、恶意、高风险的用户回复，批量测试机器人的应对能力：
1. **恶意对抗类**：「你再打我就报警」「我要投诉你骚扰」「你们是骗子吧」
2. **质疑身份类**：「你是谁？我凭什么相信你？」「你工号多少？让你们领导来跟我说」「把你们公司资质发我看看」
3. **逻辑陷阱类**：「你说我欠了100万，你把凭证发过来我看看」「我上次已经还了，你们怎么还来要？」「你利率超过法定标准了，我要告你们」
4. **极端抗拒类**：全程沉默、只说「没钱」「别打了」「我不还你能怎么样」
5. **信息错误类**：「你打错了，我不是XXX」「我从来没借过你们的钱」

#### 评估输出
- 机器人应对准确率
- 合规风险率（有没有出现违规话术、承诺不符合规定的内容）
- 系统崩溃率（遇到异常输入会不会出错）

### 🔹 方案6：实时效果预测模型
**推荐指数**：⭐⭐⭐
**落地成本**：高
**真实度**：85%
**适用阶段**：线上迭代提速

#### 核心优势
把原来需要等1-2个月的回款评估周期，缩短到当天就能出结果，迭代速度提升几十倍。

#### 实现逻辑
用历史对话数据和对应的最终回款结果，训练一个效果预测模型：
- 输入：机器人的完整对话日志、用户特征、催收阶段、话术类型等
- 输出：这个用户的还款概率（0-100%）
- 准确率：≥85%，和最终实际回款率高度相关

#### 适用场景
- 快速迭代版本：每次优化完机器人，跑几千次测试，当天就能知道效果提升还是下降
- 实时质量监控：上线后每通对话结束就能自动判断质量，不需要等回款

3. **显著性检验**：需要足够大的样本量（至少数千通话）确保统计显著性

## 🏆 黄金测试数据集规范
黄金测试数据集是基于真实业务对话标注的高质量测试用例集合，是所有效果评估、回归测试、版本迭代的基准，所有标注必须严格遵循以下规范。

### 1. 数据集构成
- 规模：≥200条覆盖全场景的真实对话
- 覆盖范围：
  - 所有催收阶段：H2（宽限期前2天）、H1（宽限期前1天）、S0（实质性逾期）
  - 所有用户类型：合作型、忙碌型、协商型、沉默型、健忘型、抗拒型、借口大师
  - 所有结果类型：成功还款、展期、失败拒绝、中断对话
  - 所有边界场景：合规风险、用户投诉、极端抗拒、逻辑陷阱

### 2. 标注字段规范
每个测试用例必须包含以下标注字段：
| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| case_id | 字符串 | ✅ | 对话唯一标识，与转写文件名一致 |
| chat_group | 枚举 | ✅ | 催收阶段：H2/H1/S0 |
| result | 布尔值 | ✅ | 最终结果：true=成功拿到还款承诺，false=失败 |
| commit_time | 字符串 | ⚠️ | 约定还款时间，result为true时必填 |
| key_nodes | 对象 | ✅ | 关键节点标记 |
| compliance_check | 布尔值 | ✅ | 合规性检查：true=话术合规，false=存在违规内容 |
| description | 字符串 | ✅ | 对话简要描述 |
| remark | 字符串 | ❌ | 备注信息（可选） |

### 3. 关键节点（key_nodes）标记规范
必须对以下关键节点进行标记：
| 节点名 | 类型 | 说明 |
|--------|------|------|
| identity_confirm | 布尔值 | 用户是否确认身份 |
| purpose_clear | 布尔值 | 坐席是否清晰说明催收来意 |
| user_resistance | 字符串 | 用户抗拒类型：无/忙/没钱/质疑/辱骂/其他 |
| response_effective | 布尔值 | 坐席应对是否有效 |
| commitment_confirm | 布尔值 | 是否确认用户的还款承诺 |
| pressure_applied | 布尔值 | 是否进行了有效施压（告知逾期后果） |

### 4. 结果判定标准
- ✅ **成功**：用户明确承诺具体还款时间（包括展期、约定后续时间）
- ⚠️ **部分成功**：用户有还款意向但未给出明确时间（归为失败）
- ❌ **失败**：用户明确拒绝还款、无明确还款意向、对话中断、用户投诉

### 5. 标注质量要求
- 准确率：标注准确率必须≥99%，所有歧义用例必须经过评审

## 代理指标校准：承诺-履约偏差修复

### 问题

当前评测的"成功"定义为"状态机走到 CLOSE 且拿到承诺时间"。这是一个**代理指标**，不等于业务真值 `repay_type`（12 小时内实际还款动作）。

基于 2026-03 全月人工坐席数据，H2 阶段口头行动率 78.1% vs 实际还款率 42.1%，**假阳性率约 46%**。意味着即使评测报告显示 70% 成功率，线上实际回款率可能仅 35-40%。

### 校准方案

#### Phase 1: 建立校准基线

利用已有的 1459 条带 `repay_type` 标签的真实通话数据：

```
┌─────────────────────────────────────────────────────────┐
│  1. 用 chatbot 重放每条真实对话（用户侧用真实转写文本）    │
│  2. 记录 chatbot 判定结果（成功/失败）                     │
│  3. 对照 repay_type 真值计算：                             │
│     - 精准率 = chatbot"成功"中实际还款的占比                │
│     - 召回率 = 实际还款用户中被 chatbot"成功"命中的占比     │
│     - 校准系数 = 精准率 / 评测成功率                       │
│  4. 分阶段（H2/H1/S0）、分客群（new_flag 0/1/2）计算      │
│     各自的校准系数                                         │
└─────────────────────────────────────────────────────────┘
```

预期产出：一个校准表，例如

| 阶段 | 评测成功率 | 精准率 | 校准后预估回款率 |
|------|:---------:|:-----:|:---------------:|
| H2 | 62.5% | ~54% | ~34% |
| H1 | 83.3% | ~48% | ~40% |
| S0 | 66.7% | ~35% | ~23% |

#### Phase 2: 提取对话级预测特征

从 chatbot 对话日志中提取与真实还款相关的特征：

| 特征类别 | 具体特征 |
|---------|---------|
| 对话结构 | 总轮次、各状态停留轮数、是否经过 PUSH_FOR_TIME |
| 用户行为 | silence 次数、借口类型序列、是否提到金额异议 |
| 承诺质量 | 承诺时间是"今天"还是"明天以后"、是否经多次追问才给出 |
| 展期路径 | 是否讨论了展期、是否同意展期 |
| 情绪信号 | 是否有威胁/辱骂、是否有犹豫表达 |

#### Phase 3: 构建校准模型

```
真实对话特征 + repay_type 标签
        │
        ▼
  训练 Logistic Regression / XGBoost
  目标：给定对话特征 → 预测实际还款概率
        │
        ▼
  校准后的评测分数 = 模型预测的还款概率
  （替代原来的二分类成功/失败）
```

#### Phase 4: 模拟器注入"虚假承诺"行为

基于真实数据中的承诺-履约偏差率，在模拟器中新增行为模式：

| 客户类型 | 虚假承诺概率 | 触发条件 |
|---------|:-----------:|---------|
| cooperative | 15% | 给出承诺时间后，repay_type 实际为空 |
| negotiating | 30% | 展期同意后不执行 |
| resistant | 40% | 被施压后口头同意，实际不付 |
| excuse_master | 50% | 借口耗尽后假意承诺 |

#### 实施优先级

| 阶段 | 依赖 | 预计收益 |
|------|------|---------|
| Phase 1 校准基线 | 1459 条真实数据 + ASR 转写（管道进行中） | 立即获得真实世界的校准系数 |
| Phase 2 特征提取 | Phase 1 完成 | 明确哪些对话行为真正影响回款 |
| Phase 3 校准模型 | Phase 2 完成 | 评测分数从 70% 相对值 → 可对标线上回款率的绝对值 |
| Phase 4 模拟器升级 | Phase 1 完成 | 模拟器可以产出更真实的成功率预估 |
