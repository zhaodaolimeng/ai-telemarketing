# 黄金数据集标注快速参考 Checklist

## 🔍 标注前检查
- [ ] 确认对话文件可以正常读取，没有损坏
- [ ] 确认speaker标注正确（AGENT=坐席，CUSTOMER=用户）
- [ ] 通读完整对话，了解整体上下文

## 📝 基本信息标注 (basic_info)
### 催收阶段 (collection_stage)
- [ ] **H2**: 刚逾期，首次催收
- [ ] **H1**: 逾期30天内，非首次催收
- [ ] **S0**: 逾期30天以上

### 通话结果 (call_result)
- [ ] **success**: 用户明确承诺具体还款时间/金额，没有异议
- [ ] **extension**: 双方达成一致，用户延期/分期还款
- [ ] **partial_payment**: 用户明确同意先还部分欠款，并给出了部分还款时间
- [ ] **failure**: 用户拒绝还款/挂断电话/没有明确还款承诺/表示无力偿还

### 通话时长 (call_duration)
- [ ] 直接使用文件中的时长数值，不需要修改

## 👤 用户画像标注 (user_profile)
### 用户类型 (persona)
- [ ] **cooperative**: 配合型，态度友好，愿意还款
- [ ] **busy**: 忙碌型，现在不方便，要求稍后联系
- [ ] **negotiating**: 协商型，主动要求延期/分期/减免
- [ ] **resistant**: 抗拒型，态度恶劣，拒绝还款，要求不要再联系
- [ ] **silent**: 沉默型，很少说话，只有简单应答
- [ ] **forgetful**: 健忘型，不记得有这笔借款，需要确认
- [ ] **excuse_master**: 借口型，有各种理由说明暂时无法还款
- [ ] **scam_suspect**: 怀疑诈骗型，质疑坐席身份、要求验证合法性
- [ ] **app_uninstalled**: APP卸载型，说已经卸载APP、无法通过APP还款
- [ ] **knowledge_seeker**: 询问型，询问很多细节问题（费用、后果、方式等）
- [ ] **short_extension_seeker**: 短期延期型，只要求宽限几天，不需要正式展期

### 抗拒程度 (resistance_level)
- [ ] **very_low**: 非常配合，主动表示会还款
- [ ] **low**: 比较配合，没有明显抗拒
- [ ] **medium**: 有一定抗拒，需要引导才同意还款
- [ ] **high**: 比较抗拒，不愿意还款，要求延期
- [ ] **very_high**: 非常抗拒，态度恶劣，明确拒绝还款，要求不要再联系

## 💬 对话轮次标注 (dialogue)
### 通用字段
- [ ] **turn_number**: 轮次序号，从1开始，自动递增
- [ ] **speaker**: agent/customer，和原始文件保持一致
- [ ] **text**: 对话内容，和原始文件保持一致
- [ ] **stage**: 对话阶段标注（见下表）

### 对话阶段 (stage) 标注
- [ ] **greeting**: 问候阶段，开场寒暄
- [ ] **identity**: 身份确认，坐席自我介绍，确认用户身份
- [ ] **purpose**: 说明来意，告知用户逾期情况
- [ ] **ask_time**: 询问用户预计还款时间
- [ ] **push**: 施压阶段，催促用户尽快还款，说明逾期后果
- [ ] **negotiate**: 协商阶段，双方讨论还款方案，用户要求延期/减免
- [ ] **handle_objection**: 处理异议阶段，回复用户疑问、安抚情绪、处理投诉威胁
- [ ] **commit**: 确认承诺，用户同意还款，确认还款时间/金额
- [ ] **close**: 结束通话，道别

### 坐席轮次 (speaker=agent) 专属字段
- [ ] **is_correct**: 回复是否正确合理
  - ✅ true: 回复合规，符合话术要求，应对正确
  - ❌ false: 回复有问题（违规、答非所问、应对不当、话术不规范等）
- [ ] **standard_response**: 当is_correct=false时，填写标准正确回复
- [ ] **user_intent**: 留空，不需要填写

### 用户轮次 (speaker=customer) 专属字段
- [ ] **user_intent**: 选择标准意图枚举值（参考标注指南中的意图列表），不要自由描述
- [ ] **is_correct**: 留空，不需要填写
- [ ] **standard_response**: 留空，不需要填写

## ⚖️ 合规检查标注 (compliance)
- [ ] **has_violation**: 对话中是否有违规内容
  - ✅ true: 存在违规
  - ❌ false: 没有违规
- [ ] **violation_details**: 当has_violation=true时，详细描述违规内容和违规点

### 常见违规类型
- 辱骂用户，使用不文明用语
- 威胁用户（上门催收、联系家人朋友、爆通讯录、报警等）
- 泄露用户隐私，在第三人在场的情况下讨论欠款
- 虚假承诺（减免利息、删除不良记录等，没有权限的承诺）
- 冒充公检法、银行等其他机构人员
- 诱导用户以贷还贷，承诺礼品返现等
- 在不合适时间催收（晚9点-早8点）
- 重复拨打骚扰用户
- 询问无关个人信息

## ℹ️ 标注信息 (annotation_info)
- [ ] **labeler**: 标注人姓名/ID
- [ ] **label_time**: 标注时间，格式 YYYY-MM-DD HH:MM:SS
- [ ] **notes**: 其他备注信息，特殊情况说明

## ✅ 标注完成检查
- [ ] 所有必填字段都已填写，没有遗漏
- [ ] 对话阶段标注准确，符合上下文
- [ ] 用户画像和抗拒程度判断一致
- [ ] 坐席回复正确性判断合理，标准回复符合要求
- [ ] 违规内容识别完整，描述清晰
- [ ] 用户意图都使用了标准枚举值，没有自由描述
- [ ] 自动标注的结果已经全部复核，错误已经修正
- [ ] 生成的JSON文件格式正确，可以正常解析

## ⚠️ 注意事项
1. 基于对话内容客观判断，不要加入主观臆测
2. 对于有歧义的地方，在notes中注明
3. 发现原始语音识别错误，在notes中说明正确内容
4. 保持标注口径一致，相同场景判断标准统一
