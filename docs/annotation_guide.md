# 黄金数据集标注指南

## 概述
本文档描述如何对印尼语催收对话进行标注，生成黄金测试数据集。该数据集将用于：
1. 对话回放测试，确保机器人优化后不出现功能退化
2. 评估指标校准，确保离线评估结果与真实业务效果一致
3. 模型训练数据，用于提升机器人的应对能力

## 标注流程

### 1. 环境准备
- 确保已经安装Python 3.8+
- 确保代码仓库已经克隆到本地
- 确保`data/processed/transcripts/`目录下有所有对话文件
- 确保`data/gold_dataset_annotation_list.json`文件已经生成

### 2. 启动标注工具
```bash
python scripts/annotation_tool.py
```

### 3. 标注步骤
1. 工具启动后会显示当前标注进度
2. 选择要标注的样本编号（优先选择高优先级的稀有场景样本）
3. 工具会显示完整的对话内容
4. 按照提示依次标注各个字段
5. 确认信息无误后保存标注结果

## 字段说明

### 基本信息 (basic_info)
| 字段 | 类型 | 说明 | 可选值 |
|------|------|------|--------|
| collection_stage | enum | 催收阶段 | H2: 刚逾期<br>H1: 逾期30天内<br>S0: 逾期30天以上 |
| call_duration | number | 通话总时长，单位秒 | 自动填充，无需修改 |
| call_result | enum | 通话结果 | success: 成功拿到还款承诺<br>failure: 催收失败<br>extension: 用户同意展期 |

### 用户画像 (user_profile)
| 字段 | 类型 | 说明 | 可选值 |
|------|------|------|--------|
| persona | enum | 用户画像类型 | cooperative: 合作型，愿意配合还款<br>busy: 忙碌型，现在没空接电话<br>negotiating: 协商型，想要商量还款方案<br>resistant: 抗拒型，不愿意还款<br>silent: 沉默型，很少说话<br>forgetful: 健忘型，不记得有借款<br>excuse_master: 借口型，有各种理由不还款 |
| resistance_level | enum | 用户抗拒程度 | very_low: 非常配合<br>low: 比较配合<br>medium: 中等抗拒<br>high: 比较抗拒<br>very_high: 非常抗拒<br>* 系统会提供建议值，可以根据实际情况调整 |

### 对话轮次 (dialogue)
每个对话轮次需要标注以下字段：
| 字段 | 类型 | 适用角色 | 说明 |
|------|------|----------|------|
| turn_number | number | 所有 | 对话轮次，从1开始，自动填充 |
| speaker | enum | 所有 | 说话人：agent(坐席)/customer(用户)，自动填充 |
| text | string | 所有 | 说话内容，自动填充 |
| stage | enum | 所有 | 对话阶段：<br>greeting: 问候阶段<br>identity: 身份确认阶段<br>purpose: 说明来意阶段<br>ask_time: 询问还款时间阶段<br>push: 施压催促阶段<br>negotiate: 协商还款阶段<br>commit: 确认承诺阶段<br>close: 结束通话阶段 |
| is_correct | boolean | 坐席 | 坐席回复是否正确合理：<br>true: 回复正确，符合催收规范<br>false: 回复不正确，违反规范或有更好的表达方式 |
| standard_response | string | 坐席 | 该场景下的标准回复，当is_correct为false时必须填写 |
| user_intent | string | 用户 | 用户说话的核心意图，简明扼要描述用户想表达的意思 |

### 合规检查 (compliance)
| 字段 | 类型 | 说明 |
|------|------|------|
| has_violation | boolean | 对话中是否有违规内容：<br>true: 有违规<br>false: 无违规 |
| violation_details | string | 违规详情，当has_violation为true时必须填写，描述具体的违规内容 |

### 标注信息 (annotation_info)
| 字段 | 类型 | 说明 |
|------|------|------|
| labeler | string | 标注人姓名 |
| label_time | string | 标注时间，自动填充 |
| notes | string | 其他备注信息，可选 |

## 标注规范

### 1. 通话结果判定标准
- **success**: 用户明确承诺在某个具体时间还款（包括承诺还款金额）
- **extension**: 用户明确要求延期还款，或者坐席同意用户延期还款
- **failure**: 其他情况，包括用户拒绝还款、用户说没钱、用户挂电话、用户说会考虑但没给具体时间等

### 2. 用户画像判定标准
| 用户类型 | 判定标准 |
|----------|----------|
| cooperative | 用户态度友好，愿意配合，没有表现出抗拒，很快承诺还款 |
| busy | 用户明确说现在很忙、在开会、在开车，让稍后再打过来 |
| negotiating | 用户主动提出想要延期、分期、减免利息，或者跟坐席商量还款方案 |
| resistant | 用户态度恶劣，明确说不还钱，让不要再来电话，或者威胁要投诉 |
| silent | 用户大部分时间不说话，只有简单的嗯啊回应，或者全程沉默 |
| forgetful | 用户说不记得有这笔借款，需要确认，或者问什么时候借的、借了多少 |
| excuse_master | 用户找各种理由不还款：没钱、家人生病、生意失败、工资没发等 |

### 3. 对话阶段判定标准
| 阶段 | 判定标准 |
|------|----------|
| greeting | 通话开始时的问候语，比如"你好"、"早上好"、"请问是XXX先生吗" |
| identity | 坐席自我介绍，说明来自哪个公司，确认用户身份的阶段 |
| purpose | 坐席说明来电目的，提醒用户有账单逾期的阶段 |
| ask_time | 坐席询问用户什么时候能够还款的阶段 |
| push | 坐席向用户施压，说明逾期后果、催促用户尽快还款的阶段 |
| negotiate | 双方协商还款方案的阶段，包括讨论延期、分期、减免等 |
| commit | 用户做出还款承诺，或者坐席确认用户还款时间的阶段 |
| close | 通话结束前的道别语，比如"谢谢配合"、"再见"等 |

### 4. 坐席回复正确性判定标准
坐席回复不正确的情况包括但不限于：
- 违反合规要求：辱骂用户、威胁用户、泄露用户隐私、虚假承诺等
- 话术不规范：没有使用礼貌用语、没有称呼用户、表达不清晰等
- 逻辑错误：回答用户问题错误、给出错误的信息、前后矛盾等
- 应对不当：面对用户的抗拒或疑问没有采取合适的应对策略

当回复不正确时，需要提供标准的正确回复作为参考。

### 5. 用户意图标注规范
- 简明扼要，用一句话描述用户的核心意图
- 不需要逐字翻译，提炼核心意思即可
- 比如用户说"我现在很忙，你晚点再打过来"，意图可以标注为"用户现在很忙，要求稍后再联系"
- 比如用户说"我这个月没钱，能不能下个月再还"，意图可以标注为"用户要求延期一个月还款"

### 6. 违规判定标准
违规内容包括但不限于：
- 辱骂用户：使用侮辱性、攻击性语言
- 威胁用户：威胁要上门催收、联系家人朋友、爆通讯录、起诉等
- 泄露隐私：在有第三人的场合讨论用户的欠款情况
- 虚假承诺：承诺减免利息、删除不良记录等没有权限的承诺
- 冒充其他机构人员：冒充公检法、银行工作人员等
- 诱导性表述：承诺还款送礼品、返现等
- 不合适时间催收：在晚上9点到早上8点之间催收
- 重复拨打骚扰：短时间内多次拨打用户电话
- 询问无关信息：询问与还款无关的个人信息
- 使用不文明用语：脏话、粗口等

### 7. 语言规范
**所有标注内容必须统一使用英文**，包括但不限于：
- 对话阶段(stage)字段描述
- 用户画像(persona)字段描述
- 用户意图(user_intent)标注
- 违规详情(violation_details)描述
- 备注(notes)字段内容
- 所有标注相关的说明性文字
目的是确保数据集的国际化兼容性，方便后续多语言模型训练和全球团队协作。

## 注意事项
1. 标注时以对话内容为准，不要受系统建议值的影响
2. 对于有歧义的地方，可以在备注中说明
3. 如果发现对话转录错误，可以在备注中说明正确的内容
4. 标注过程中随时可以按Ctrl+C中断，已标注的内容不会自动保存
5. 标注完成后请仔细检查，确认无误后再保存
